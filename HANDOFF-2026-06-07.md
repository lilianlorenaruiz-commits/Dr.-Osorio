### Nota técnica — bug del preview MCP

Durante la verificación del Bloque 5 (confirmation panel) se detectó que el browser de testing del preview MCP no dispara el evento `close` del `<dialog>` cuando se llama `dialog.close()` programáticamente. En browsers reales (Chrome/Firefox/Safari/Edge actuales) este evento SÍ dispara síncronamente por spec. El cleanup del modal funciona correctamente en producción; el test programático requería `dispatchEvent(new Event('close'))` para simular el flow real. **Workaround para futuros tests:** después de `dialog.close()` en preview, hacer `dialog.dispatchEvent(new Event('close'))` para que el listener corra.

### Workflow lesson — cache busting del JS

Cuando se hacen refactors no triviales en `js/main.js` (que es shared cross-page), el browser puede cachear la versión anterior aunque el HTML pida reload. **Solución:** versionar el `<script src="js/main.js?v=N">` igual que el CSS. Aplicado en `for-attorneys.html` con `?v=3`. Considerar replicar en home y for-individuals si se modifica `main.js` nuevamente.
